# 📦 Courier Tracking System — Spring Boot REST API

A beginner-friendly Spring Boot application demonstrating clean layered architecture,
REST API design, JPA database integration, validation, and global exception handling.

---

## 🏗️ Architecture Overview

```
Client (Postman / Browser)
        │
        ▼  HTTP Request
┌─────────────────────┐
│   CourierController  │  ← @RestController  (Layer 1: HTTP)
│  /courier endpoints  │    Parses JSON, validates, delegates
└────────┬────────────┘
         │  method call
         ▼
┌─────────────────────┐
│   CourierService     │  ← @Service         (Layer 2: Business Logic)
│  createCourier()     │    UUID generation, status rules, domain logic
│  getByTracking()     │
│  updateStatus()      │
└────────┬────────────┘
         │  JPA calls
         ▼
┌─────────────────────┐
│  CourierRepository   │  ← @Repository      (Layer 3: Data)
│  JpaRepository       │    SQL auto-generated by Spring Data JPA
└────────┬────────────┘
         │  SQL
         ▼
┌─────────────────────┐
│   H2 In-Memory DB    │  ← Database         (H2 / can swap to MySQL)
└─────────────────────┘
```

## 🚀 How to Run

### Option 1: Using Maven Wrapper (recommended)

```bash
# 1. Navigate to the project root (where pom.xml lives)
cd courier-tracking-system

# 2. Build and run
./mvnw spring-boot:run          # macOS / Linux
mvnw.cmd spring-boot:run        # Windows
```

### Option 2: Build JAR and Run

```bash
# Build
mvn clean package -DskipTests

# Run
java -jar target/courier-tracking-system-1.0.0.jar
```

### Option 3: Import into IDE (IntelliJ / Eclipse)

1. Open IntelliJ IDEA → File → Open → select the `courier-tracking-system` folder
2. IntelliJ detects `pom.xml` automatically and downloads dependencies
3. Open `CourierTrackingSystemApplication.java`
4. Click the green ▶ Run button

2. Fill in:

   - **User Name:** `sa`
   - **Password:** *(leave blank)*
3. Click **Connect**
4. Run SQL to inspect data:

```sql
-- See all couriers
SELECT * FROM COURIER;

-- Count by status
SELECT STATUS, COUNT(*) FROM COURIER GROUP BY STATUS;

-- Find specific courier
SELECT * FROM COURIER WHERE TRACKING_NUMBER = 'TRK-XXXXXXXX';
```

## 📮 API Reference + Postman Testing Guide

### Base URL: `http://localhost:8080`

---

### ① POST /courier — Create Shipment

| Field      | Value                    |
|------------|--------------------------|
| Method     | POST                     |
| URL        | `http://localhost:8080/courier` |
| Body Type  | raw → JSON               |

**Request Body:**
```json
{
    "senderName":   "Alice Smith",
    "receiverName": "Bob Jones",
    "source":       "Mumbai",
    "destination":  "Delhi"
}
```

**✅ Success Response (HTTP 201 Created):**
```json
{
    "id": 1,
    "trackingNumber": "TRK-3F2A1B4C",
    "senderName": "Alice Smith",
    "receiverName": "Bob Jones",
    "source": "Mumbai",
    "destination": "Delhi",
    "status": "CREATED",
    "createdAt": "2024-04-01T10:30:00"
}
```

**❌ Validation Error (HTTP 400):**
```json
// Body with blank senderName: { "senderName": "", ... }
{
    "status": 400,
    "error": "Validation Failed",
    "message": "One or more fields have invalid values.",
    "fieldErrors": {
        "senderName": "Sender name is required"
    }
}
```

---

### ② GET /courier/{trackingNumber} — Track Shipment

| Field      | Value                                          |
|------------|------------------------------------------------|
| Method     | GET                                            |
| URL        | `http://localhost:8080/courier/TRK-3F2A1B4C`   |

**✅ Success Response (HTTP 200 OK):**
```json
{
    "id": 1,
    "trackingNumber": "TRK-3F2A1B4C",
    "senderName": "Alice Smith",
    "receiverName": "Bob Jones",
    "source": "Mumbai",
    "destination": "Delhi",
    "status": "CREATED",
    "createdAt": "2024-04-01T10:30:00"
}
```

**❌ Not Found (HTTP 404):**
```json
{
    "timestamp": "2024-04-01T10:31:00",
    "status": 404,
    "error": "Not Found",
    "message": "Courier not found with trackingNumber: TRK-UNKNOWN",
    "path": "/courier/TRK-UNKNOWN"
}
```

---

### ③ PUT /courier/{id}/status — Update Status

| Field      | Value                               |
|------------|-------------------------------------|
| Method     | PUT                                 |
| URL        | `http://localhost:8080/courier/1/status` |
| Body Type  | raw → JSON                          |

**Request Body:**
```json
{
    "status": "IN_TRANSIT"
}
```

**Valid status values:**
- `CREATED`
- `PICKED_UP`
- `IN_TRANSIT`
- `OUT_FOR_DELIVERY`
- `DELIVERED`
- `CANCELLED`

**✅ Success Response (HTTP 200 OK):**
```json
{
    "id": 1,
    "trackingNumber": "TRK-3F2A1B4C",
    "senderName": "Alice Smith",
    "receiverName": "Bob Jones",
    "source": "Mumbai",
    "destination": "Delhi",
    "status": "IN_TRANSIT",
    "createdAt": "2024-04-01T10:30:00"
}
```

**❌ Invalid Enum (HTTP 400):**
```json
// Body: { "status": "FLYING" }
{
    "status": 400,
    "error": "Bad Request",
    "message": "Invalid JSON or enum value. Valid status values: ..."
}
```

## 🧪 Full Postman Test Flow

Follow these steps in order to simulate a complete shipment lifecycle:

```
Step 1: POST   /courier              → Create shipment         → Status: CREATED
Step 2: GET    /courier/{trackingNumber}  → Confirm creation
Step 3: PUT    /courier/1/status     → { "status": "PICKED_UP" }
Step 4: PUT    /courier/1/status     → { "status": "IN_TRANSIT" }
Step 5: PUT    /courier/1/status     → { "status": "OUT_FOR_DELIVERY" }
Step 6: PUT    /courier/1/status     → { "status": "DELIVERED" }
Step 7: GET    /courier/{trackingNumber}  → Confirm final status
Step 8: Check  H2 console            → SELECT * FROM COURIER;
```

## 🔑 Key Concepts Demonstrated

| Concept                  | Where                                  |
|--------------------------|----------------------------------------|
| REST API Design          | `CourierController.java`               |
| Layered Architecture     | controller → service → repository      |
| JPA Entity Mapping       | `Courier.java` with `@Entity`, `@Column` |

| H2 Console               | `application.properties`              |

## 🔄 Switching to MySQL (optional)

1. Replace H2 dependency in `pom.xml` with:
```xml
<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
</dependency>
```

2. Update `application.properties`:
```properties
spring.datasource.url=jdbc:mysql://localhost:3306/courier_db?createDatabaseIfNotExist=true
spring.datasource.username=root
spring.datasource.password=your_password
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.jpa.database-platform=org.hibernate.dialect.MySQLDialect
```
